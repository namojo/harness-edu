# AGENTS.md — Codex 엔지니어링 공통 규율

Codex는 작업을 시작할 때 이 파일을 자동으로 읽는다. 여기에 적힌 규칙은 매 프롬프트에
다시 쓰지 않아도 항상 적용된다 — 그것이 규칙을 파일로 두는 이유다.

## 불변 규칙 (모든 작업 공통)

- **최소 권한**: 분석·리뷰·조사는 읽기 전용으로 수행한다. 파일을 고쳐야 할 때만
  쓰기 권한을 쓰고, 무엇을 왜 바꿨는지 diff로 설명한다.
- **근거 우선**: 코드 변경은 "무엇을 왜"를 함께 남긴다. 수치·주장에는 출처를 붙이고,
  확인 못 한 내용은 지어내지 말고 "[확인 필요]"로 표시한다.
- **검증 없는 완료 금지**: 코드를 고쳤으면 테스트/실행으로 확인하기 전까지 "완료"라고
  말하지 않는다. 테스트가 없으면 그 사실을 보고한다.
- **검증자 분리**: 구현한 주체가 자기 결과의 최종 승인자가 되지 않는다. 리뷰는
  `codex review` 또는 reviewer 에이전트로 교차 검증한다.
- **재현성**: 반복 작업은 프로파일(`-p`)과 스키마(`--output-schema`)를 써서 매번 같은
  형식으로 산출한다.

## 산출물 규약

- 코드 변경: 반드시 변경 요약(무엇을·왜)과 함께 제시. 테스트 결과 첨부.
- 리포트·분석: 최종 결과는 `--output-last-message`로 분리 가능한 단일 마크다운/JSON으로.
- 구조화 데이터: 스키마가 있으면 그 형태를 지킨다. 임의 필드 추가·누락 금지.

## 문체

- 개조식 평서체. 과장("혁명적", "완벽")·불확실을 감춘 단정 금지.
- 코드 주석과 커밋 메시지는 "무엇"이 아니라 "왜"를 적는다 (무엇은 diff가 이미 말한다).
