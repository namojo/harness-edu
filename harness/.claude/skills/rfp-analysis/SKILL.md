# RFP 요구사항 전수 추출 방법론

제안서 실패의 첫째 원인은 문장력이 아니라 **누락**이다. 이 스킬은 누락을
구조적으로 차단하는 추출 절차를 정의한다.

## 1. 원문 확보 (형식별)
- **hwpx**: zip 컨테이너다. 압축 해제 후 `Contents/section0.xml`의 `<hp:t>` 텍스트를
  문단(`<hp:p>`) 단위로 추출한다. `Preview/PrvText.txt`와 분량을 대조해 누락을 확인한다.
- **pdf/docx**: 각 전용 스킬의 추출 절차를 따른다.
- 표가 깨진 채 추출되면 반드시 재시도한다 — ETL 범위표 같은 핵심이 표에 있다.

## 2. ID 체계

| 접두 | 대상 | 예 |
|---|---|---|
| PRJ | 사업 개요(명칭·기간·목적) | PRJ-03 기간 60일 |
| REQ | 기능·비기능 요구 | REQ-07 전송 모니터링 |
| DOM | 데이터·업무 범위 | DOM-05 진단정보(KCD-8) |
| CON | 조건·의무 조항 | CON-06 원격개발 불가 |
| DEL | 산출물 | DEL-02 품질진단 결과보고서 |

## 3. 암묵 요구 도출 규칙 (누락의 주범)
- **목적·배경 문단**을 스캔하라: 과업내역에 없는 요구가 목적에만 등장한다
  (예: "가명화 ETL"). 발견 즉시 REQ로 승격한다.
- **기대효과 문단**을 스캔하라: "~체계 수립"은 요구의 완곡 표현이다
  (예: "보안 연계망 구축").
- **표의 각주·말미 문장**을 스캔하라: "반드시 ~할 것"은 강행 규정이다.
  강행 규정에는 [강행] 표식을 붙인다.

## 4. 리스크 표식
원가·법무·일정에 영향을 주는 조항에 ← 화살표 경고를 붙인다:
초단기 일정, 유지보수 의무 기간, 소스 귀속, 원격 금지, HW/SW 부담, 특허 조항.

## 5. 인벤토리 형식
`artifacts/요구사항_인벤토리.md`에 [개요/기능/범위/조건/산출물/배경지식] 섹션으로
작성한다. 각 항목은 `ID | 내용 | (근거 위치)` 를 지킨다. 이 파일이 팀의 유일한
기준표이며, 제안서 추적 매트릭스와 검수 기준표가 모두 여기서 파생된다.
