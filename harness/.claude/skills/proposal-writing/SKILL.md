# 공공 제안서 작성 표준

## 문서 구조 (순서 고정)
제안 요약 → 1.사업의 이해 → 2.요구사항 추적 매트릭스 → 3.기술 제안 →
4.사업 관리(WBS·조직·리스크) → 5.과업 조건 전건 수용 확약 → 6.산출물·검수 기준 →
7.유지보수 계획 → 맺음말

## 핵심 장치 3종 (없으면 미완성)
1. **추적 매트릭스(2장)**: [ID | 과업지시서 요구 | 제안 대응 | 제안서 절] 4열 표.
   요구사항 인벤토리의 REQ 전건이 행으로 존재해야 한다. 이 표는 검수 기준표로
   재사용된다고 명시한다.
2. **조건 수용 확약표(5장)**: CON 전건에 대해 [ID | 조항 | 수용·이행 방안].
   "수용"만 쓰지 말고 이행 방안(어느 장, 어떤 체계)을 병기한다.
3. **산출물-검수 기준표(6장)**: DEL 전건 + 검수 기준(안). 공공 관례상
   사업수행계획서·완료보고서 등 추가 제출물을 함께 제안한다.

## 문체 규칙
- 개조식 평서체. 과장 형용사("최고의", "혁신적") 금지.
- 확약은 수치로, 수치에는 이행 방안을 병기: "미매핑률 ≤1% — 미매핑 대장 상시
  집계와 W3 확정 회의로 관리".
- 발주처가 확인할 수 없는 문장("최선을 다하겠습니다")은 삭제 대상이다.
- 사실과 해석을 분리한다.

## 일정 표현
- "1단계 약 3주" 금지. 주 단위 표(W1~Wn)로 쓰고 임계경로와 마일스톤 산출물을
  명시한다.

## docx 생성
docx-js 사용, 한글 폰트 Malgun Gothic. 표는 columnWidths+셀 width(DXA) 이중 지정,
음영은 ShadingType.CLEAR. 생성 후 PDF 렌더링 검수를 수행한다.
