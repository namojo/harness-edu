# 보건의료 데이터 표준 지식 사전

과업지시서에는 표준의 **이름만** 등장한다. 이 사전이 이름을 실무 지식으로 바꾼다.
여기 없는 항목은 웹 검색으로 보강하되 반드시 출처를 남기고, 검증 후 이 사전에 추가한다.

## 표준코드 체계

| 표준 | 정체 | 적용 대상 | 매핑 실무 포인트 |
|---|---|---|---|
| FHIR R4 / KR Core | HL7 국제 의료정보 교환 표준의 국내 프로파일 | 상태값 전반(내원상태, 처방상태·의도, 검사상태, 알레르기 등) | 원천 코드 → FHIR ValueSet 매핑표를 만들어 발주처 확정. 허용값 외 코드는 품질진단 검출 대상 |
| KCD-8 | 한국표준질병사인분류 8차 | 진단명 | 원천에 KCD-7 구코드 잔존 가능 → 전환표 적용, 미전환 건 별도 보고 |
| 심평원 EDI | 수가·약가·치료재료 청구 코드 | 수술·처치명, 약품제품명, 각종 검사명 | 병원 원내 자체코드가 흔함 → 원내-EDI 매핑테이블 필요(보험심사팀 협조가 지름길) |
| ATC | WHO 해부·치료·화학 분류 | 약품 성분, 알레르기 원인물질 | 제품코드→주성분→ATC 2단계 매핑 |
| 심평원 기관코드 | 요양기관번호·종별·진료과목·보험자 | 의료기관 정보 | 고시 코드표 직대조(오매핑 위험 낮음) |
| 질병청·복지부 | 예방접종명(질병청)·KD코드·의사 면허코드 | 예방접종, 진료의 | 기관 고시 기준, 개정 이력 추적 필요 |

## 가명화 법적 준거
- **개인정보보호법 제28조의2**: 과학적 연구 등 목적의 가명정보 처리 근거.
- **보건의료데이터 활용 가이드라인(복지부)**: 항목별 가명처리 방법
  (직접 식별자 삭제·대체, 준식별자 처리 기준).
- **비정형 데이터**(판독문·경과기록·병리결과 자유텍스트): 텍스트 내 이름·등록번호·
  연락처가 숨어 있다. 실무 표준은 정규식·사전 규칙 + 개체명인식(NER) 결합 검출이며,
  자동 검출만으로는 부족하므로 **층화 표본 사람 검증**을 병행한다.

## 사업 맥락
- **국가통합바이오빅데이터 구축사업**: 범부처(복지부·과기부·질병청 등) 사업.
  참여자(동의 기반 등록자)의 임상정보·유전체를 수집하며, 의료기관은 표준 스키마로
  임상정보를 공급하는 관문 역할을 한다. 병원 시스템의 품질이 곧 국가 데이터 품질이다.
- **병원 EMR 현실**: 원천 스키마는 병원·벤더별로 다르고 문서화가 부실한 경우가
  많다 → 실데이터 프로파일링 기반 역공학과 병원 전산팀 합동 분석이 필수.
