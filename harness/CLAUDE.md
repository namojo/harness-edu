# CLAUDE.md — 공공 제안서 작성 하네스

## 프로젝트 컨텍스트
나라장터 과업지시서(`과업지시서_데이터뱅크.hwpx`)를 입력으로 사업 제안서(docx)를
작성하는 프로젝트다. 발주기관은 양산부산대학교병원, 과업은 국가통합바이오빅데이터
EMR 연계 ETL 시스템 구축(60일)이다.

## 불변 규칙 (모든 에이전트 공통)

- **전수 대응 원칙**: 과업지시서의 모든 조항은 ID(REQ/DOM/CON/DEL)가 부여되어야
  하며, 제안서의 어느 절이 어떻게 대응하는지 추적 가능해야 한다. 대응 없는 조항이
  하나라도 남으면 미완성이다.
- **근거 없는 확약 금지**: 수치 확약(예: 미매핑률 ≤1%)은 이행 방안이 함께 서술될
  때만 쓴다. "최선을 다하겠다"류의 검증 불가능한 문장은 쓰지 않는다.
- **임의 해석 금지**: 과업지시서가 모호한 지점(예: 표준 스키마의 정확한 버전)은
  지어내지 말고 "발주처 협의 확정" 절차로 명시한다.
- **조건 조항은 원가·법무 리스크**: 6장 조건 조항(유지보수·소스 귀속·원격개발
  불가·HW/SW 부담)은 반드시 전건 수용 여부와 이행 방안을 표로 명시한다.
- **문체**: 공공 제안서 관례(개조식 평서체, 과장 금지)를 따른다. 상세는
  proposal-writing 스킬 참조.

## 산출물 위치
- 중간 산출물: `artifacts/` (요구사항 인벤토리, 매핑 전략 노트 등)
- 최종 제안서: 프로젝트 루트 `제안서_*.docx`

## 실행 모드
에이전트 팀을 기본으로 하며, 파이프라인 순서는
rfp-analyst → domain-expert → (solution-architect ∥ pm-expert) → proposal-writer 통합
→ compliance-reviewer 검증이다. compliance-reviewer의 지적은 해당 담당 에이전트가
수정한다.
